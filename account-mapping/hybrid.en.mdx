## What Is a Hybrid Account?

A hybrid account has **different taxonomy mappings** for the current year and the previous year. This is used when an account's classification has changed between reporting periods.

## Create a Hybrid Account

<Steps>
  <Step title="Open the account editor">
    Click the three-dot menu (⋯) on an account card and select **"Edit"**, or select **"Hybrid Account"** directly.
  </Step>
  <Step title="Enable hybrid mode">
    Toggle the **"Hybrid Account"** switch in the editor dialog.
  </Step>
  <Step title="Assign categories">
    Select a **Current Year Mapping** category and a different **Previous Year Mapping** category.
  </Step>
  <Step title="Save">
    Confirm the mapping. The account will appear in both taxonomy positions in the mapped panel.
  </Step>
</Steps>

<Note>
The hybrid account option is only available when both current year and previous year values exist for the account.
</Note>

## Visual Indicator

Unmapped account cards show a **hybrid icon** when one year has already been mapped but the other hasn't. Clicking the icon navigates to the mapped year in the taxonomy tree.

## Create and Edit Accounts

You can also create new accounts or edit existing ones:

1. Click **"Add account"** in the actions menu, or **"Edit"** on an existing account
2. Enter the **Name**, **Number**, **Current Year**, and **Previous Year** values
3. Optionally enable **Hybrid Account** and select separate category mappings
4. Click **Save**

<Warning>
Account numbers must be unique. If a number is already in use, you will see the error: "Account number already in use."
</Warning>
