## WebSocket

- Path: `/anexo/ws/socket.io`
- Protocol: WebSocket (wss/ws)
- Auth: Token-based
- Reconnection: Max 5 attempts, 1-5s exponential backoff
- Multi-tab sync via BroadcastChannel API

### Shared-State Patch Contract

Communication between main and popout windows uses a typed patch protocol for state synchronization. There are two modes:

**Legacy mode (default):** Sends the full state as a snapshot on every update. All fields are transmitted with every message.

**Incremental mode** (enabled via `incrementalSharedStatePatch` feature flag): Sends only changed fields as delta updates. Each message contains:

| Field | Type | Description |
|---|---|---|
| `contractVersion` | `1` | Protocol version |
| `contractMode` | `'incremental'` | Identifies incremental messages |
| `referencesDelta` | object? | Delta for document references with `reset`, `version`, `documents`, and `changedDocumentIds` |
| `selectionDelta` | object? | Delta for selection state (focused, selected, and edited annotations) |
| `highlightDelta` | object? | Delta for highlight state (hover and marked annotations) |
| `numbersDelta` | object? | Delta for numbers audit state |

Scalar fields (e.g., `viewMode`, `activePageIndex`, `mode`) are transmitted directly at the top level of the message in both modes.

Additionally, the protocol supports action updates for commands such as `findAndChangeDocumentTab`, `updatePageIndex`, `resetInteractionMode`, and `closeFileManagerDialog`.
