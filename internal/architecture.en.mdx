## Microservices Architecture

Notes Auditor consists of several specialized Python services:

| Service | Port | Python | Description |
|---|---|---|---|
| **anexo** | 2846 | 3.10 | Main backend (FastAPI) — API, business logic, auth |
| **external_api** | 2855 | 3.10 | Public API for external integrations |
| **delphi** | 3211 | 3.11 | LLM-based document analysis with RAG (Qdrant) |
| **spectra** | 5379 | 3.12 | Number detection and attribute extraction |
| **mentat** | 3212 | 3.11 | Account mapping (6-stage matching) |
| **document_converter** | 4741 | 3.9 | DOCX→PDF conversion (LibreOffice) |

## Checklist Versioning

Checklists support multiple versions (e.g., `initial`, `2026_01`, `2026_02`). Each checklist has an **active version** that determines which questions and control questions are used for new assignments.

- **Version resolution:** When creating an assignment, the active version of the checklist is stored. Existing assignments retain their version.
- **Auto-upgrade:** If an upgrade chain is configured, existing assignments are automatically migrated to the newer version on next access. Answers for unchanged questions are preserved; new questions are added as unanswered.
- **Environment-specific versions:** A different version can be configured per deployment environment (e.g., production, test).

## Communication

- **REST API** — Synchronous client-server communication
- **WebSocket (Socket.IO)** — Real-time updates (assignment status, file processing, AI analysis)
- **Huey (Redis)** — Asynchronous task queue (I/O-intensive + CPU-intensive)
- **Aioredlock** — Distributed locks for concurrency control
