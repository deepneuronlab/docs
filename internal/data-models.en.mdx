| Field | Type | Description |
|---|---|---|
| assignment_id | FK | Associated assignment |
| pdf_id | FK | Document |
| question_id | FK | Linked question |

`start_offset` and `end_offset` are persisted in the annotation `custom_data` at creation time and during re-referencing. Historic references without offsets are back-filled lazily on read or via the `backfill_reference_offsets_task`.
</Note>

### DocumentDiff (Version Comparison)
| Field | Type | Description |
|---|---|---|
| new_document_id | string | ID of the newer document |
| old_document_id | string | ID of the older document |
| state | enum | Processing status: `not_started`, `in_progress`, `ready`, `failed` |
| new_annotations | Map | Annotations in the newer document |
| old_annotations | Map | Annotations in the older document |

Each diff annotation contains:
- **kind**: `added`, `deleted`, or `unchanged`
- **classification**: `editorial`, `substantive_expected`, or `substantive_unexpected`
- **annotation**: Position (pageIndex, boundingBox, rects), text, and linked IDs

### AnnotationDeltaPayload

Represents an incremental update to PDF annotations for a single document, used for efficient real-time synchronization instead of transmitting the full annotation array.

| Field | Type | Description |
|---|---|---|
| revision | number | Monotonically increasing revision counter per document |
| mode | `"incremental"` \| `"rebuild"` | `incremental` applies only the listed changes; `rebuild` replaces all annotations |
| toCreate | AnnotationText[] | Annotations to add |
| toUpdate | AnnotationText[] | Annotations to modify |
| toDelete | AnnotationText[] | Annotations to remove |

`AnnotationDeltasByDocument` is a `Record<string, AnnotationDeltaPayload | undefined>` that maps document IDs to their delta payload. It is transmitted as the `annotationDeltas` field inside the shared-state WebSocket update.
