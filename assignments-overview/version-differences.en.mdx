## New Versions of a Document

A new version of an audit document or a supporting document can be uploaded to the assignment via the **File Manager.**

By clicking **"Upload Documents"** in the top right corner of the File Manager, the user can choose to add a **new version** of an **audit document** or a **supporting document.** Next to all listed documents, the button **"Upload a new version"** allows adding a document directly from the computer.

The app automatically highlights differences between document versions. With the tab **"View Version Differences"** above the document view, you can enter the compare flow to review changes side by side.

### Compare flow

When you open the compare flow, the app displays both document versions side by side with synchronized scrolling. If multiple document pairs are available, you first select which pair to compare.

Differences are classified into three categories:

- **Editorial** — Minor wording or formatting changes
- **Substantive (expected)** — Material changes that were anticipated
- **Substantive (unexpected)** — Material changes that require closer review

You can filter the displayed annotations by classification using the filter controls in the compare toolbar. Annotations are highlighted in both document panes, and clicking an annotation scrolls the opposite pane to the corresponding change.

<Note>
The diff comparison may take a moment to process. A loading indicator is shown while the comparison is being prepared. The processing status progresses through `not_started`, `in_progress`, and `ready`. If processing fails, the system automatically attempts to regenerate the comparison. During regeneration, the status returns to `in_progress`. If regeneration cannot be attempted (for example, within one hour of a previous attempt), the status is set to `failed` and you can retry later.
</Note>

This functionality ensures that all changes between document versions are clearly visible and can be reviewed.

After uploading a new document version, some references may no longer be found. In this case, the question in the audit overview will have the status **"re-check,"** indicated by a **purple** flag. If the reference is recognized in the document, it will be marked in **blue**.

The upload of a new document version has a direct effect on the progress that has been made in the checklist. After uploading a new version of an audit document, all answered questions will be set back to AI-proposal.

<Warning>
**Note:** The new version upload must be confirmed by clicking **"Confirm"** in the bottom right corner of the window. Before confirming, it is still possible to delete the uploaded version. After confirmation, the newly added version cannot be deleted, and the audit will automatically be updated and rechecked.
</Warning>
